基于hexo + github免费图床的博客搭建后，如何被想要订阅的人看到？

没错
就是使用rss订阅

  1. 在hexo中，需要先安装对应的插件
  2. 在配置文件_config.yml配置如下内容 这是一个常见的RSS Feed配置，你可以根据你的需求进行修改。其中：
  • type 定义了Feed的类型，可以是 ‘atom’、’rss2’ 或者 ‘json’。
  • path 定义了Feed文件的输出路径。
  • limit 定义了Feed中的文章数目限制。
  • order_by 定义了Feed中文章的排序方式。
  • autodiscovery 为 true 时，Hexo 会在 <head> 中插入一个 auto-discovery 标签。
  1. 清空缓存文件
    hexo clean
  • 清除缓存文件 (db.json) 和已生成的静态文件 (public)。Hexo会在生成新的静态文件之前缓存旧的静态文件，以便更快地生成新的文件。如果你在开发过程中遇到了问题，或者在部署前想确保一切都是最新的，那么可以使用hexo clean命令。
  1. 重新生成静态文件
    hexo generate
  • 命令会将你的Markdown文件转换为静态HTML文件，以便在网页服务器上托管。生成的静态文件通常位于项目的public目录中。
  1. 部署网站
    hexo deploy
  • 根据你在Hexo的配置文件_config.yml中设置的部署方法，该命令会自动部署你的博客。比如，如果你设置了GitHub Pages作为部署方法，那么Hexo就会将生成的静态文件推送到你的GitHub仓库。

步骤3-5可以简写为：hexo clean && hexo g -d

  1. 在rss的订阅软件中，填入以下url来订阅博客
    https://你的博客域名/atom.xml
